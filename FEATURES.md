# 视频人物替换工具 - 功能与流程说明

## 一、功能概述

本工具用于对原视频中的**指定时间片段**做**人物替换/动效合成**，整体流程为：

1. 前端按时间轴选择多个片段。
2. 后端先把这些片段从原视频中裁剪并合并成一条短视频。
3. 把短视频和人物正面图一起提交给阿里云百炼进行视频生成。
4. 将百炼返回的新视频按片段时长再次切分，对应替换回原视频中的原片段位置，输出一条完整成片。

当前只提供「正式处理」（调用百炼）的模式。

---

## 二、前端核心能力

### 1. 媒体加载与布局

- **左侧区域**：上方为视频预览，下方为时间轴；右侧为片段列表与处理按钮。
- **人物图上传**：左下角区域单独上传人物正面图，上传后会自动走 `/api/upload` 存到 OSS，并在设置中使用。

### 2. 时间轴与缩放

- **可见范围**：通过 `visibleStart ~ visibleEnd` 控制当前视野，仅显示整片视频的一小段，支持长视频精细操作。
- **缩放方式**：
  - 时间轴/标尺上滚轮缩放，缩放锚点为白色播放头。
  - 顶部缩放滑块（减号/加号按钮 + 拖拽 knob），支持从整片到「逐帧」级别的放大。
- **逐帧级别**：
  - 放到最大时，时间轴可以精确到**每一帧**，并显示 `1f / 2f / 3f ...` 这样的帧号。
  - 同时每帧约 18 像素宽，能够看清每一帧的缩略图。

### 3. 时间轴交互

- **悬停预览**：鼠标在时间轴左右滑动时，左侧视频会跳转到对应时间并暂停预览，白色竖线跟随鼠标，便于「擦片」找帧。
- **点击设入点 / 出点**：
  - 第一次点击时间轴：设为入点，提示「已设入点 x:xx.xx，点击设出点」。
  - 再点击一次：设为出点，自动按入点/出点生成片段加入列表。
  - 设入点后，时间轴上会用半透明区块实时预览入点→当前鼠标位置的区间。
- **拖拽框选**：
  - 直接在时间轴按下左键拖拽，松开时生成一个片段（长度 ≥ 0.05 秒）加入列表。
- **Space + 左键平移视野**：
  - 按住空格再按住左键拖动，可左右平移时间轴视野（不改变播放时间），且不触发单击逻辑。
  - 拖动时播放头始终对齐当前鼠标时间点。
- **点击标尺**：
  - 在时间轴上方标尺栏点击，会把播放头和视频时间跳到对应位置，并自动让该位置居中显示。

### 4. 片段列表

- 列表中每一项展示：
  - `开始时间 - 结束时间（本段时长 X.X 秒）`
- 支持：
  - 删除单条片段。
  - 底部展示「片段总时长：X.X 秒」。
  - 片段总时长需在 2~30 秒之间，才允许开始处理或测试。

### 5. 设置与状态展示

- **API Key 设置**：
  - 设置面板中填写阿里云百炼的 API Key，支持保存到 `localStorage`。
  - 提供「获取 API Key」和「账户充值」的快捷链接。
- **进度与提示**：
  - 顶部右侧展示进度条文字：上传并处理中、测试中、处理完成等。
  - 所有错误通过统一的中文文案弹窗展示，不会直接把接口的英文错误/日志展示给用户。

## 三、后端处理流程（简版）

### 1. 参数校验

- 路由：`POST /api/process`。
- 校验内容：
  - 必须有上传视频、有效的 `segments` 数组和 `duration`。
  - 合并片段总时长需在 2~30 秒之间。
  - 正式模式下必须有 `apiKey` 与人物图的 `personOssUrl`。

### 2. 按片段合并成短视频

- 使用 `segments`（单位秒）从原视频中按顺序裁剪出多个片段，拼接为一条新视频 `merged-*.mp4`。
- 时间全部使用秒的小数，统一保留 3 位小数，避免浮点误差。

### 3. 调用阿里云百炼（正式模式）

- 把 `merged-*.mp4` 和人物图传到 OSS，创建百炼任务，并轮询任务状态：
  - 成功：拿到 `video_url`。
  - 失败或 10 分钟内未完成：返回「任务失败」或「任务超时」的中文错误。
- 下载百炼返回的视频为 `result-*.mp4`。

### 4. 规范化合成结果（关键优化）

- 对百炼返回的 `result-*.mp4` 先跑一遍规范化：
  - 固定输出为 30fps，重新编码为规整的 H.264 + AAC。
  - 生成 `normalized-*.mp4`，作为后续切分和替换的输入。
- 若规范化失败，则退回使用 `result-*.mp4` 继续流程。

### 5. 按片段时长切分合成结果

- 对规范化后的合成结果视频（优先 `normalized-*.mp4` 否则 `result-*.mp4`）：
  - 按原始 `segments` 的每段时长，从 0 开始累加 `t`，依次用 `-ss t -t duration` 切成多段 `repl-*.mp4`。
  - 若合成结果总时长略短于片段总时长，最后一段自动缩短为剩余可用时长。

### 6. 替换回原视频

- 构建一条完整时间线：`原片前段 + repl-0 + 原片中段 + repl-1 + ... + 原片尾部`。
- 视频部分：
  - 对原视频各区间用 `trim start=/end=` 裁剪，对替换段直接引用各 `repl-*.mp4`。
  - 所有段 concat 后，再统一做 `fps=30` 输出，避免重复帧过多导致编码卡死。
- 音频部分：
  - 默认直接从原视频整条 trim 到 `duration`，确保音轨连续。
  - 若音频映射有问题，会自动退回到「仅替换视频、不带音频」的无声模式。

## 四、错误处理与用户提示

- **百炼任务超时**：提示「百炼任务超过 10 分钟仍未完成，可能队列繁忙或片段总时长过长，请适当缩短片段后重试。」  
- **替换阶段超时**：提示「替换合并阶段超过 120 秒仍未完成，已自动终止，请尝试缩短视频或片段总时长后重试。」  
- **视频时间戳异常 / 重复帧过多**：提示「检测到视频时间戳异常（重复帧过多），建议重新导出源视频或缩短片段后再试。」  
- 其它错误：统一包一层「处理失败：{错误信息}」，避免直接暴露底层日志。
